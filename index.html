Disena la Landing de una App con Sass

Te entregamos la landing de una app ficticia (FocusFlow) en HTML sin ningun estilo. Tu mision: convertirla en una landing page atractiva escribiendo el CSS con Sass -variables, nesting, compilacion y minificacion-, personalizarla y publicarla con GitHub Pages.

[ ]Punto de partida

Trabajaras sobre el archivo index.html (carpeta proyecto-base/ o copialo del bloque de abajo). Abrelo en el navegador: se ve sin estilos, porque aun no existe el CSS. Ese es tu lienzo.

  • El HTML ya trae la estructura y las clases listas (.nav, .hero, .feature, .plan, .testimonio, .cta, .footer...).
  • No reescribas la estructura ni metas estilos dentro del HTML: todo el diseno va en tu archivo Sass.
  • Si puedes (y debes) agregar contenido nuevo en las zonas marcadas con ZONA DE PERSONALIZACION.

</>El HTML que vas a usar

Crea un archivo index.html y pega este codigo. Es tu punto de partida. No le agregues estilos aqui: solo completaras las zonas marcadas y todo el diseno ira en tu Sass.

>Tu mision

Crea scss/estilos.scss, escribe ahi TODOS los estilos de la landing y compilalo para generar el CSS que usa el HTML. La pagina debe quedar moderna y ordenada: navegacion, hero llamativo, grilla de funciones, planes de precios (con el plan Pro destacado), opiniones, banner final y footer.

Pondras en practica: VariablesNestingCompilarMinificarPersonalizacionGitHub Pages

[v]Requisitos tecnicos obligatorios

Tu Sass debe cumplir las cuatro condiciones. (Aqui va el "que"; el "como" lo resuelves con lo visto en clase.)

1

Variables

Define al menos 6 variables (colores de marca, fuente, espaciados, radio...) y usalas en toda la landing. Debe poder cambiar de aspecto editando solo las variables.

2

Nesting

Estiliza los componentes con anidamiento que refleje el HTML. Usa & para :hover y para los modificadores (&--destacado, &--primario). Maximo 3 niveles.

3

Compilar

Compila tu .scss a css/estilos.css y comprueba que la landing se ve bien. Entrega fuente y compilado.

4

Minificar

Genera tambien css/estilos.min.css (produccion) y comparala con la normal. Incluye ambas en el repo.

Como minimo deben quedar estilizados: la navegacion, el hero con sus botones, la grilla de funciones, los planes de precios (destaca el Pro), las opiniones, el banner CTA y el footer.

*Tu parte para personalizar (obligatoria)

Haz que esta landing sea tuya. Ademas de estilizar, debes:

  • Inventa tu propia app: cambia el nombre "FocusFlow", el lema y los textos por una idea tuya (real o ficticia).
  • Tu propia funcion: agrega al menos una cuarta tarjeta en la grilla de .features.
  • Una seccion nueva inventada por ti: en la zona marcada, crea una seccion que no existia (ej. Preguntas frecuentes, Estadisticas, Descarga la app), inventa sus clases y estilizala.
  • Tu identidad visual: elige la paleta y la tipografia de tu marca mediante variables.

Reto extra (opcional): ofrece dos temas (claro/oscuro) cambiando solo los valores de las variables.

^Publicacion: repositorio + GitHub Pages

  1. Crea un repositorio publico nuevo en tu cuenta de GitHub.
  2. Sube tu proyecto completo (con index.html en la raiz y las carpetas scss/ y css/).
  3. Entra a Settings -> Pages y activa GitHub Pages usando la rama principal (main) y la carpeta raiz.
  4. Abre la URL publica que entrega GitHub y verifica que la landing se ve con todos sus estilos.
Si en la pagina publicada se ve "sin estilos", casi siempre es por la ruta del CSS en el <link> o porque olvidaste subir la carpeta css/.

=Checklist de entrega

  • Estilice navegacion, hero, funciones, precios, opiniones, CTA y footer.
  • Defini y use al menos 6 variables; el diseno cambia editando solo las variables.
  • Use nesting que refleja el HTML y & para hover/modificadores (max. 3 niveles).
  • El plan Pro se ve destacado respecto a los demas.
  • Compile a css/estilos.css y el HTML lo carga correctamente.
  • Genere la version minificada css/estilos.min.css.
  • Cambie el nombre/textos por mi propia app.
  • Agregue mi propia tarjeta de funcion.
  • Cree e integre una seccion nueva inventada por mi.
  • Subi el proyecto a un repositorio publico de GitHub.
  • Publique y verifique la landing en GitHub Pages.

iPistas (sin soluciones)

  • Abre el HTML sin estilos primero: asi entiendes la estructura antes de pintar.
  • Compila en modo watch para ver tus cambios al instante.
  • Recuerda: nunca edites el .css, edita el .scss y recompila.
  • Empieza por las variables (tu paleta) y estiliza componente por componente.
  • Documentacion: sass-lang.com/documentation - docs de GitHub Pages.

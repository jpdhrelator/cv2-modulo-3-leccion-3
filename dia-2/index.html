Estiliza un Dashboard con Mixins, Partials y 7-1

Te entregamos el HTML de un panel de administracion (dashboard) sin estilos. Tu mision: estilizarlo con Sass aplicando lo del dia 2 -mixins, @content, partials con @use, patron 7-1 y funciones/@each-, personalizarlo y publicarlo en GitHub Pages.

[ ]Punto de partida

Trabajaras sobre el index.html que esta mas abajo (copialo del bloque). Se ve sin estilos porque aun no existe el CSS. Tu objetivo es organizar el Sass en varios archivos y compilarlo a css/main.css.

  • El HTML ya trae las clases listas (.layout, .sidebar, .topbar, .stat, .badge, .tabla, .boton...).
  • No metas estilos en el HTML: todo va en tus archivos Sass.
  • Completa las zonas marcadas con ZONA DE PERSONALIZACION.

[v]Requisitos tecnicos obligatorios

Lo que debes aplicar (del dia 2). El "que"; el "como" lo resuelves con lo visto en clase.

1

Partials + @use

Divide tu Sass en varios partials (_archivo.scss) y reunelos en un main.scss con @use. Compila SOLO main.scss.

2

Patron 7-1

Organiza los partials en carpetas al estilo 7-1 (al menos abstracts/, base/, components/, layout/).

3

Mixins

Crea y usa al menos 2 mixins: uno con argumentos (ej. boton/badge por color) y uno con @content para una media query responsive.

4

Funciones / @each

Usa @each (o @for) para generar las variantes de color de .badge (exito, alerta, error) a partir de un mapa.

5

Variables + namespace

Centraliza colores/espaciados en abstracts/ y usalos con @use ... as (namespace) desde los demas archivos.

6

Compilar + minificar

Genera css/main.css y tambien css/main.min.css (produccion).

Como minimo deben quedar estilizados: el layout (sidebar + contenido), la topbar con botones, las tarjetas de estadistica, los badges de colores, la tabla y el estado activo del menu.

1.2.3Como abordarlo: proceso paso a paso

Sigue este orden. En cada paso vas a tomar decisiones de diseno: no hay una unica respuesta correcta, pero si decisiones mas coherentes. Piensa antes de escribir codigo.

  1. Observa y planifica. Abre el HTML sin estilos e identifica los bloques: layout, sidebar, topbar, tarjetas de estadistica, tabla y badges. Decide la sensacion general: panel sobrio y profesional, o moderno y colorido.
  2. Define tu paleta (en abstracts/). Elige: un color primario (acciones, enlace activo, boton principal), un acento para detalles, los neutros (fondo de pagina, fondo de tarjetas, color de texto y un gris para texto secundario) y 3 colores de estado para los badges: exito, alerta y error.
    Preguntate: ¿hay suficiente contraste para leer el texto sobre cada color?
  3. Tipografia y espaciado. Decide la fuente de titulos y la de texto, una escala de espaciados (pequeno / medio / grande) y un radio de borde que repetiras en tarjetas, botones y badges. La consistencia es lo que hace que se vea "profesional".
  4. Base. Aplica un reset minimo y los estilos del body (fondo y color de texto de tu paleta). A partir de aqui todo hereda.
  5. Layout. Decide como repartir .sidebar y .contenido (ancho del sidebar, que el contenido ocupe el resto) y el color del sidebar. ¿Oscuro para contrastar, o claro y minimalista?
  6. Sidebar y menu. Estiliza el logo y los enlaces. Decide como se ve el enlace activo (--activo) frente a los normales y que pasa en hover.
  7. Topbar. Titulo + botones. Cuida la jerarquia: el boton primario debe destacar mas que el secundario.
  8. Componentes reutilizables. Estiliza botones (variantes primario/secundario), badges (sus 3 estados), tarjetas de estadistica (sombra, padding, como resalta el valor) y la tabla (cabecera, filas, alternancia/hover).
    Preguntate: ¿que propiedades se repiten entre estos? Eso es candidato a mixin.
  9. Estados e interaccion. Define hover en botones, enlaces y filas; un foco visible y el cursor adecuado. Los detalles de interaccion dan sensacion de calidad.
  10. Responsive. Decide que cambia en pantallas chicas (¿el sidebar se apila arriba? ¿las stats pasan a una columna?) y aplicalo con tu mixin @content.
  11. Personaliza. Cambia la marca, agrega tu tarjeta y tu widget propio (ver seccion siguiente).
  12. Compila, revisa y publica. Mira el resultado en el navegador, ajusta, minifica y sube a GitHub Pages.
Consejo: trabaja componente por componente y compilando en modo watch. Es mas facil que intentar estilizar todo de una vez.

EJEjemplo guiado para arrancar

Para que no partas de cero, aqui tienes un ejemplo completo del patron con UN solo componente (el boton): defines una variable, creas un mixin, lo usas en el componente y lo reunes en main.scss. Replica esta misma idea para el resto: sidebar, topbar, stats, tabla y badges.

ejemplo de arranque (Sass)
// archivo: scss/abstracts/_variables.scss
$primario: rgb(32, 156, 238);
$texto:    rgb(40, 40, 40);
$radio:    8px;


// archivo: scss/abstracts/_mixins.scss
// Mixin con argumento: recibe el color de fondo y arma el boton.
@mixin boton($fondo) {
  background: $fondo;
  color: white;
  border: 0;
  padding: 0.6rem 1.2rem;
  border-radius: 8px;
  cursor: pointer;
}


// archivo: scss/components/_boton.scss
// Trae las variables y el mixin con @use (namespace v. y m.)
@use '../abstracts/variables' as v;
@use '../abstracts/mixins' as m;

.boton {
  @include m.boton(v.$primario);          // boton primario (azul)
  &--secundario { @include m.boton(v.$texto); }   // variante con &
}


// archivo: scss/main.scss
// El main SOLO reune los partials con @use. Compila SOLO este archivo.
@use 'components/boton';

Para verlo funcionar, compila con la extension Live Sass Compiler de VS Code (la que vimos en clase): abre main.scss y pulsa Watch Sass en la barra inferior. Se generara css/main.css automaticamente cada vez que guardes. Recuerda que los _partials no se compilan solos; solo el main.

Ese es todo el flujo: variable -> mixin -> componente -> main. Ahora aplicalo al resto. Pista para los badges: en vez de escribir 3 reglas casi iguales, recorre un mapa de colores con @each (lo viste en clase).

*Tu parte para personalizar (obligatoria)

  • Tu marca: cambia el nombre "DataPanel" y elige tu paleta desde las variables de abstracts/.
  • Tu propia tarjeta de estadistica: agrega al menos una en la zona marcada.
  • Un widget/seccion nuevo inventado por ti (ej. actividad reciente, notas, tareas): inventa sus clases y estilizalo.
  • Reto extra (opcional): agrega un mixin de tema oscuro y aplicalo cambiando solo variables.

^Publicacion: repositorio + GitHub Pages

  1. Crea un repositorio publico en GitHub.
  2. Sube el proyecto completo (con index.html en la raiz, mas scss/ y css/).
  3. Activa Settings -> Pages con la rama main y la carpeta raiz.
  4. Abre la URL publica y verifica que el dashboard se ve con estilos.
Si se ve "sin estilos", revisa la ruta de css/main.css en el <link> o que subiste la carpeta css/.
